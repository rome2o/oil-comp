## ðŸ”„ Change Request Process

### How to Request Scope Changes

**Step 1: Submit Written Request**

Use this format:
```
Change Request: [Title]
Requested by: [Name]
Date: [Date]

Description:
[What you want to change/add]

Business Justification:
[Why this is important]

Estimated Impact:
[How this affects timeline/budget]
```

**Step 2: Impact Assessment**

We evaluate:
- **Hours Required:** How many hours will this take?
- **Priority:** Is this P0 (must-have) or P2 (nice-to-have)?
- **Timeline Impact:** Does this delay other work?
- **Cost:** Does this require tier upgrade or separate billing?

**Step 3: Proposal & Approval**

We respond with:
- Feasibility assessment
- Hour estimate
- Options:
  - **Option A:** Include in current tier (if hours available)
  - **Option B:** Defer to next month
  - **Option C:** Upgrade tier
  - **Option D:** Separate project (billed separately)

**Step 4: Execute (If Approved)**

- Written approval required before proceeding
- Update roadmap and hour allocation
- Track hours against change request

## ðŸ”‘ Key Principles

### 1. Scope Protects the Program

**Narrow scope = Deep impact**
- Better to excel at product page optimization than be mediocre at everything
- Focus = Results

### 2. Transparency = Trust

**No hidden "that's out-of-scope" surprises**
- Everything documented upfront
- Gray areas clarified proactively
- You know what you're paying for

### 3. Flexibility Within Boundaries

**We're not rigid:**
- Reprioritization within scope: Always okay
- Minor adjustments: Usually accommodated
- Major changes: Discussed openly

**But we have boundaries:**
- Out-of-scope = separate approval
- Over-cap = explicit agreement
- Rush work = premium pricing

### 4. Business Justification Required

**Every request evaluated by impact:**
- Will this drive revenue?
- Does this support the core goal (add-to-cart optimization)?
- Is this the highest-priority use of hours?

**Low-impact requests politely declined:**
> "We could do that, but it won't move the needle on add-to-cart rate. Instead, let's focus on [higher-impact task]."
